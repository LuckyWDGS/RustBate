# UnrealEditor 项目架构与源码详解 (新手指南)

欢迎进入桌面软件开发的世界！这是一个基于 **Tauri** 框架开发的桌面应用程序。Tauri 的特点是：
- **前端 (Vue 3 + TypeScript)**：负责绘制你看到的精美界面（各种按钮、弹窗、项目卡片等）。
- **后端 (Rust)**：负责和操作系统打交道（比如扫描本地硬盘的文件夹、运行额外的 `exe` 程序等），它相当于应用程序的大脑底层。

这份文档旨在帮助初学者快速理解本项目中各个文件的作用，以及前后端是如何协同工作的。

---

## 1. 核心文件与目录结构解析

项目根目录下主要分为两大块：前端代码和后端代码。

### 1.1 前端部分 (UI 界面与交互逻辑)

前端负责展示页面，接收用户的点击，处理相关的展示逻辑。

- **`src/`**：核心的前端代码都在这里。
  - **`App.vue`**：**最核心的前端文件！** 你看到的整个软件界面，以及你在界面上的搜索、点击、弹窗操作，全都在这个文件里。它主要分为三个部分：
    1. `<template>`：用 HTML 写的页面结构（比如顶部搜索栏、项目卡片列表、克隆项目的弹窗）。
    2. `<script setup lang="ts">`：用 TypeScript 写的交互逻辑（比如点击“扫描项目”按钮后，应该去请求哪个后端目录的数据）。在这个区域，你会看到很多针对后端功能的调用操作。
    3. `<style scoped>`：用 CSS 写的界面样式（按钮颜色、各种框的大小、悬浮动画等）。
  - **`main.ts`**：前端配置入口文件。它的作用是把刚才说的网页应用挂载到系统窗口内的一块“空画板”上。
  - **`assets/`**：存放前端界面需要用到的小图标或静态图片。
- **`index.html`**：前端网页的基石。软件启动后，浏览器内核首先加载这个界面骨架，再通过它引入 `src/main.ts`。
- **`package.json`**：记录前端开发中使用了哪些第三方库（比如 Vue）以及项目的各种启动/打包命令（比如平时用的 `npm run dev`）。
- **`vite.config.ts`**：Vite 前端构建工具的配置文件，控制前端代码如何被编译打包，通常你不需要去修改它。
- **`tsconfig.json`**：TypeScript 的配置文件，用来做强制代码类型检查的。

### 1.2 后端部分 (Rust 系统操作)

涉及操作系统和底层的核心操作（如扫文件、启动虚幻引擎），必须交给后端（Rust）来执行。因为只有这样，应用才能突破网页安全限制，自由读取电脑文件。

- **`src-tauri/`**：这个文件夹存放所有的后端代码、打包设置和权限文件。
  - **`src/lib.rs`**：**最核心的后端文件！** 里面写满了一个个带有 `#[tauri::command]` 标记的 Rust 函数，这些标记代表这些函数是专门**暴露给前端调用的接口**。具体核心功能有：
    - `scan_projects`：帮前端扫描指定的文件夹下有哪些 `.uproject` 后缀的文件，获取相关信息。
    - `get_templates`：扫描你的模板路径，获取所有可用模板信息，供新建项目功能使用。
    - `run_project`：使用 Windows PowerShell 脚本后台启动指定的 UE 项目。
    - `rename_project`：给项目重命名。不只是改 `.uproject` 文件的名字，还要负责改外层根文件夹的名字。
    - `clone_project`：将选定的项目/模板目录完整地复制一份保存到新位置，并自动重命名内部全部文件。
  - **`src/main.rs`**：Rust 程序的入口，只做简单的启动和初始化工作，把我们在 `lib.rs` 里写的全部交互功能挂载起来执行。
  - **`tauri.conf.json`**：Tauri 的核心配置文件。如果你要修改打包后软件的图标是什么、叫什么名字、启动窗口多大等外观特性，都在这里改。
  - **`capabilities/default.json`**：权限配置文件。用来告诉系统，前端被允许调用哪些原生的权限（例如，本项目中它声明了弹出系统文件夹选择框、读取特定外部进程等安全权限）。

## 2. 程序的运行流程：前端与后端是如何交互的？

为了让你更好地理解前端和 Rust 之间如何互打电话，以界面上点击**“扫描项目”**功能为例：

1. **用户操作** (前端)：你在界面的输入框中填入路径（如 `D:/Projects`），并点击“扫描项目”按钮。
2. **向后端发送请求** (前端)：此时 `App.vue` 中的 `scanProjects` 函数开始执行。它利用了一句特殊的语句 `invoke('scan_projects', { dirPath: scanPath.value })`，相当于前台员工给后台发了个消息指令：“去查一下这个路径里有哪些项目！”。
3. **执行系统级操作** (后端)：Rust 后台里的 `src-tauri/src/lib.rs` 文件收到了同名 `scan_projects` 的命令和你要查的路径。Rust 极其擅长高速文件操作，它会在这个路径下遍历探索，找到所有 `.uproject` 文件，并把项目名称、预览图（`ProjectPreview.png`）和最近更新时间打包起来。
4. **返回数据** (后端)：Rust 将搜查到的所有信息整理成一个详细列表并返回给发命令的前端。
5. **渲染更新界面** (前端)：前端 `App.vue` 接收到这个数据列表后，利用 Vue 框架的核心武器“数据响应式绑定”，瞬间将这一个个项目的封面画出变成供你点击的项目可视化大卡片。

## 3. 以后的修改指南与学习建议

在以后你需要添加新功能或自己修改代码时，可以参考下面这三条“速查路线”：

1. **想要换颜色、调整字体大小或改变排版？**
   - > **去哪改：** 打开 `src/App.vue`，划到底部的 `<style scoped>` 区域里修改 CSS 规则。这部分的修改非常直观，绝不会弄坏核心逻辑功能。

2. **想要增加提示词、改变确认窗口内容等交互逻辑？**
   - > **去哪改：** 同样打开 `src/App.vue`，在中间 `<script setup lang="ts">` 区域寻找你要修改的函数（比如你想改新建类的报错提示，可以搜 `showToast`）。修改这里使用的是 JavaScript/TypeScript 操作语言。

3. **想要增加一个系统级的全新底层功能？**（比如：删除选中的项目，或者读取某个 TXT 文件中的配置内容）
   - > **这是个“两步走”战略：**
     1. 首先去后端 `src-tauri/src/lib.rs` 里用 Rust 写一个能在电脑里安全删文件/读文件的新功能。写好后一定要加 `#[tauri::command]` 标签，且别忘登记到下方的 `builder` 接口内。
     2. 接着回到前端 `src/App.vue` 中，用 `invoke('你写的命令名称')` 来发起呼叫即可。
